## **객체**

-   클래스는 객체가 아니다.
    -   클래스는 객체를 만들기 위한 청사진이다.
-   인스턴스란 객체를 부르는 다른 이름이다.

<br>

#### 객체의 상태와 행동

클래스에서는 객체가 아는 것과 객체가 하는 것을 기술한다.

-   인스턴수 변수 (상태) ➡️ 아는 것
-   메소드 (행동) ➡️ 하는 것

<br>

## **메소드 매개변수**

메소드에서 사용하는 것은 매개변수(parameter), 호출하는 쪽에서 전달하는 것은 인자(argument)라고 한다.

-   매개변수는 메소드 본체 안에서만 사용되기 때문에 지역 변수이다.
-   인자는 2개 이상도 전달 가능하다.
-   자바에서는 값으로 전달한다. 즉, 복사본을 전달한다.

```
class Foo {
	void go(int x) {
		x = 0;
	}
}

int x = 7;
foo.go(x); // x가 0이 되지 않는다.
```

<br>

## **캡슐화(Encapsulation)**

클래스 내 인스턴스 변수와 메소드를 캡슐처럼 감싸 외부에 직접 접근하지 못하도록 은닉하여 보호하는 것

-   캡슐화를 하지 않은 경우, 점 연산자를 사용하여 클래스의 인스턴스 변수에 직접 접근 가능하다.
    -   theCat.height = 0;
    -   인스턴수 변수를 직접 접근하면, 변수 값을 부적절하게 설정하는 경우를 방지하기 힘들기 때문에 보호가 필요하다.
-   다른 코드에서 클래스 내의 인스턴수 변수에 직접 접근할 수 없도록, getter / setter를 사용한다. (정보 은닉)
-   인스턴스 변수는 private으로 지정하고, 접근 제어를 위한 getter / setter는 public으로 지정한다. 

<br>

#### 인스턴스 변수와 지역 변수 차이

-   인스턴스 변수는 클래스 내에서 선언된다.
-   지역 변수는 메소드 내에서 선언된다.
    -   지역 변수는 기본값이 없으므로 사용 전에 반드시 초기화해야 한다.
-   메소드 매개변수는 메소드 내(메소드의 인자 목록 내)에서 선언되므로 지역 변수와 거의 같다.
    -   하지만 인자로 전달될 때 항상 값이 설정되므로 초기화가 되지 않는 경우는 없다.

<br>

## **객체의 동치**

힙에 객체가 두개 들어있고, 두 레퍼런스가 각 객체를 참조하지만 그 두 객체가 동치인 것으로 간주할 수 있는 경우

<br>

#### 변수 비교

**\==**

-   물리적 동치성 비교
-   단순한 비트 비교 ➡️ 비트 패턴이 같으면 true
-   변수 크기에 대해서는 신경쓰지 않는다.
-   원시 유형 두 개를 비교하거나, 레퍼런스 두 개가 같은 객체를 참조하는지 확인할 때 사용한다.

```
int a = 3;
byte b = 3;
if (a == b) // true
```

**equals()**

-   논리적 동치성 비교
-   서로 다른 두 객체가 똑같은지 확인할 때 사용한다.
